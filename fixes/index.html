2

Staff search field does nothing

galCompliance.Items is set incorrectly — scrAdminDash

Formula set wrong
What happened The galCompliance Items property was set to txtAdminSearch (just the name of the text input control) instead of the full Filter formula that references it. The gallery had no idea what to display or filter.

Select galCompliance in the Tree View on scrAdminDash. Set the property dropdown to Items. Replace whatever is there with this formula:

galCompliance — Items
Items · galCompliance 
Filter(colCompliance,
    (drpTypeFilter.Selected.Value = "All" ||
     StaffType = drpTypeFilter.Selected.Value) &&
    (IsBlank(txtAdminSearch.Text) ||
     txtAdminSearch.Text in Name)
)
After this fix Typing in the search box filters the list instantly as you type — no search button needed. The type filter dropdown also works at the same time. Both filter together.
3

Compliance gallery shows nothing

scrAdminDash · drpTypeFilter · Default

Root cause found
Why it's empty — nothing to do with your data drpTypeFilter has no Default property set. When the screen loads, the dropdown's selected value is blank. The gallery Items filter checks drpTypeFilter.Selected.Value = "All" — blank does not equal "All", so that condition is false. The second condition (StaffType = drpTypeFilter.Selected.Value) checks if StaffType equals blank — also false for everyone. Every single row gets filtered out. Your 2 people are in colCompliance, they're just being hidden by the type filter every time the screen loads.

Select drpTypeFilter on scrAdminDash. Add this one property:

PropertyValue
Default"All"

That's the entire fix. The gallery will now show everyone on load, and the filter still works when you pick a specific staff type.

4

StaffType dropdown blank in people list

scrAdminManage · galPeople · drpRowType · Default

Why it's blank drpRowType.Default is set to ThisItem.StaffType. In SharePoint, StaffType is a Choice column — so ThisItem.StaffType returns a record like {Value: "Civil Servant"}, not a plain string. The dropdown Items list is a plain string array, so passing a record as Default finds no match and shows blank. The value IS there, it just can't be pre-selected with a record.

Select drpRowType inside galPeople. Change Default to:

PropertyCurrentFix
DefaultThisItem.StaffTypeThisItem.StaffType.Value
5

Recurrence dropdown blank when editing a course

scrAdminManage · drpEditRecur · Default

Why it's blank drpEditRecur has no Default set at all. When you open a course to edit, the recurrence dropdown has no way to know which option to pre-select, so it shows blank. If you save without touching it, it saves the recurrence as 0 and corrupts the course record.
Set this directly in Studio — do not paste it via YAML The Default formula contains {Label: ..., Months: ...} syntax which breaks Power Apps' own YAML parser. Click drpEditRecur in the Tree View, switch the formula bar to the Default property, and type the formula directly there.

Select drpEditRecur on scrAdminManage. Set the Default property directly in Studio — it looks up the matching row in the options table based on the current course's saved recurrence months:

Default · drpEditRecur 
LookUp(
    Table(
        {Label: "One-off (never expires)", Months: 0},
        {Label: "Every 6 months",          Months: 6},
        {Label: "Every year",              Months: 12},
        {Label: "Every 18 months",         Months: 18},
        {Label: "Every 2 years",           Months: 24},
        {Label: "Every 3 years",           Months: 36}
    ),
    Months = varEditCourse.RecurrenceMonths
)
The table in Default must exactly match the table in Items If you've added or removed options from drpEditRecur.Items, update this table to match — same Label text, same Months values, same order.

Clicking a team member shows your own training

scrTeamMember · galMyChain · recRowBG · OnSelect

Why it shows the wrong person When you tap a team member, recRowBG.OnSelect builds colViewTraining and then navigates to scrMyTraining. But scrMyTraining.OnVisible runs immediately on arrival and completely rebuilds colMyTraining using your own email (varMyEmail) — wiping out colViewTraining before the gallery ever reads it. The screen then shows your own training.

Two changes needed:

Step 1 — Update recRowBG.OnSelect in galMyChain. Find the Navigate call at the end and replace it. The full OnSelect should end with:

recRowBG.OnSelect — end of formula (replace Navigate line) 
ClearCollect(colMyTraining, colViewTraining);
ClearCollect(colMyRecords, colViewRecords);
Set(varViewingForSelf, false);
Navigate(scrMyTraining, ScreenTransition.Cover)

Step 2 — Guard scrMyTraining.OnVisible so it only rebuilds training data when viewing your own record. Wrap the existing ClearCollect block:

scrMyTraining.OnVisible — wrap existing data load 
If(
    varViewingForSelf,
    ClearCollect(colMyRecords, Filter('TT TrainingRecords', Lower(PersonEmail) = varMyEmail));
    // ... rest of your existing ClearCollect calls go here, unchanged
)
Also set varViewingForSelf back to true on the back button When the user taps back from scrMyTraining to scrTeamMember (or to their own home screen), set varViewingForSelf back to true so their own data loads correctly next time they visit scrMyTraining directly.
11

Courses gallery errors: "StatusKey does not exist"

scrAdminManage · galCourses · Items

Why it errors AddColumns(colCourses, StatusKey, ...) without quotes makes Power Apps look for an existing column called StatusKey — which doesn't exist. Adding quotes ("StatusKey") causes a different error ("Expected identifier name") due to a quirk in how Power Apps parses computed column names in this context. The fix is to drop AddColumns entirely and use nested Sort calls instead.

Click galCourses in the Tree View. Switch the formula bar to the Items property and replace the entire formula with this:

Items · galCourses 
Sort(
    Sort(colCourses, Title, SortOrder.Ascending),
    Status.Value,
    SortOrder.Ascending
)

The inner Sort orders courses A–Z by title. The outer Sort groups Active courses before Inactive ("Active" < "Inactive" alphabetically). Same visual result as the original sort — no AddColumns required.

Same pattern in scrMyTraining and scrTeamMember If you see a similar error mentioning SortKey (scrMyTraining · galMyTraining) or PersonRequired / PersonInDate (scrTeamMember · galMyChain), repaste those screens from the Screens tab — the YAML files for those screens already use the correct quoted syntax.

ModernDropdown not pre-selecting saved value

scrAdminManage · drpEditRecur · drpRowType · Default

Why the dropdown looks blank ModernDropdown uses a Default property that takes a record value like {Value: "something"}. That syntax contains : which Power Apps' YAML parser misreads as a mapping — so it can't be set via YAML paste. Set it directly in Studio instead.

drpEditRecur — recurrence when editing a course

Click drpEditRecur → formula bar → select the Default property. Type:

Default · drpEditRecur 
LookUp(
    Table(
        {Label: "One-off (never expires)", Months: 0},
        {Label: "Every 6 months",          Months: 6},
        {Label: "Every year",              Months: 12},
        {Label: "Every 18 months",         Months: 18},
        {Label: "Every 2 years",           Months: 24},
        {Label: "Every 3 years",           Months: 36}
    ),
    Months = varEditCourse.RecurrenceMonths
)

drpRowType — staff type in the People list

If you want to use ModernDropdown (not Classic) for drpRowType, click it in the gallery → formula bar → Default property. Type:

Default · drpRowType 
{Value: ThisItem.StaffType.Value}
Classic/DropDown is fine too The YAML screens use Classic/DropDown for drpRowType because it lets you write Default: =ThisItem.StaffType.Value as a plain string — no record syntax, no YAML parse error. The only difference is the visual style. If you prefer the ModernDropdown look, apply the formula above directly in Studio.
