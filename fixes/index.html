Status pill colours unreadable

scrAdminDash · galCompliance · galDrillDetail · btnDetailPill  ·  v1.0 — 21 Jun 2026

Why it looked wrong btnDetailPill was a ModernButton. ModernButton runs BasePaletteColor through Fluent's tonal palette generator — light colours come out washed-out grey, dark text on grey = unreadable. The fix changes it to a plain Label so Fill and Color are used directly, no tonal processing.

In the Tree View, expand galCompliance → expand a row → expand galDrillDetail → click btnDetailPill. Set these two properties in the formula bar:

Fill · btnDetailPill 
=Switch(ThisItem.TrainingStatus,
    "Expired",       RGBA(226,75,74,1),
    "Expiring Soon", RGBA(239,159,39,1),
    "Not Started",   RGBA(136,135,128,1),
    "Complete",      RGBA(55,138,221,1),
    RGBA(99,153,34,1))
Color · btnDetailPill 
=RGBA(255,255,255,1)
This only applies if btnDetailPill is a Label The scrAdminDash v1.9 YAML paste already replaces the control with a Label. If you haven't repasted scrAdminDash yet, paste it first — then these formulas are already baked in and you don't need to do anything manually.
