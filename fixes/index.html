1

Navigate in OnVisible

Affects: scrStart · scrAdminDash · scrAdminManage

⛔ Error

Navigate cannot be used here since it would automatically always navigate away from this screen.

Why this happens Power Apps runs OnVisible every time the screen loads. If Navigate() is inside it, the app immediately jumps away — which loads the new screen — which might redirect back — infinite loop. Power Apps blocks it entirely.

The fix is a hidden Timer control. The timer fires automatically 1 millisecond after the screen loads, then the Navigate call goes inside the timer's OnTimerEnd instead of OnVisible. The screen loads first, then the timer fires, then the navigation happens — no loop.

Important — one timer per screen You need to add a separate timer to each of the three screens: scrStart, scrAdminDash, and scrAdminManage. A timer only works on the screen it lives on. Follow steps 1–6 three times — once on each screen — then paste the correct formula for that screen in step 5.

Step-by-step — repeat on each of the three screens

1
Select the screen in the Tree View

Click the screen name (e.g. scrStart) in the left panel. Make sure the screen itself is selected, not a control inside it.

2
Insert a Timer

In the top ribbon: Insert → Timer. A white timer box will appear on the canvas. It doesn't matter where it sits — drag it to a corner or off the edge.

3
Rename the timer (optional but tidy)

In the Tree View, right-click the new timer → Rename → type tmrNav → press Enter.

4
Set these four properties on the timer

With the timer selected, use the property dropdown (top-left of the formula bar) to switch between properties and set each value:

PropertyValue to typeWhat it does
AutoStarttrueTimer starts the moment the screen loads
Duration1Fires after 1 millisecond (effectively instant)
RepeatfalseOnly fires once, not on a loop
VisiblefalseHides the timer box from users
5
Paste the Navigate formula into OnTimerEnd

In the property dropdown, select OnTimerEnd. Copy the formula for your screen from the section below and paste it in.

6
Remove Navigate from OnVisible

Click the screen (not the timer) in the Tree View. Set the property dropdown to OnVisible. Delete only the Navigate line(s). Leave any ClearCollect, Set, or other lines exactly as they are.

How to check it worked The red error underline on OnVisible should disappear. The Navigate call now sits in OnTimerEnd, which Power Apps permits.
Screen 1 of 3 — scrStart

Go to scrStart. Add a timer there following steps 1–6 above. In step 5, paste this formula into that timer's OnTimerEnd. Then delete the Navigate lines from scrStart → OnVisible.

OnTimerEnd · scrStart 
If(
    IsBlank(varMe),
    Navigate(scrNoRecord, ScreenTransition.Fade),
    Navigate(scrMyTraining, ScreenTransition.Fade)
)
Screen 2 of 3 — scrAdminDash

Go to scrAdminDash. Add a second, separate timer to this screen following steps 1–6. In step 5, paste this formula into that timer's OnTimerEnd. The ClearCollect calls already in OnVisible stay exactly where they are — only the Navigate line gets removed.

OnTimerEnd · scrAdminDash 
If(!varIsAdmin, Navigate(scrMyTraining, ScreenTransition.None))
Screen 3 of 3 — scrAdminManage

Go to scrAdminManage. Add a third, separate timer to this screen following steps 1–6. In step 5, paste this formula into that timer's OnTimerEnd. All ClearCollect and other lines stay in OnVisible — only the Navigate line gets removed.

OnTimerEnd · scrAdminManage 
If(!varIsAdmin, Navigate(scrMyTraining, ScreenTransition.None))

Sort on Choice column

Affects: galCourses Items · scrAdminManage

⛔ Error

Cannot sort on the expression type.

Why this happens The Status column is a Choice column (a preset dropdown, not plain text). SortByColumns() only works on text and number columns — it cannot sort Choice fields directly.
Replacement formula

Select galCourses in the Tree View. Set the property dropdown to Items. Replace the entire current formula with this:

Items · galCourses 
Sort(
    Sort(colCourses, Title, SortOrder.Ascending),
    Status.Value,
    SortOrder.Descending
)

This uses Sort() (not SortByColumns()) and reads Status.Value — the text label of the choice — which is sortable. The inner sort applies Title A→Z first, then the outer sort layers Status on top.

If Status.Value still errors Try Text(Status) in place of Status.Value. This depends on whether Status is a Dataverse Choice or a SharePoint Choice column — both have slightly different Power Fx representations.

Delegation warning

Affects: btnRetry · scrNoRecord

⚠ Warning — no action needed

The "Lower" part of this formula might not work correctly on large data sets.

This is a yellow warning, not a red error. The app will build and run fine.

What it means LookUp('TT Personnel', Lower(Email) = varMyEmail) uses Lower() to compare emails case-insensitively. Power Apps can't push Lower() to the data source, so it fetches records locally and filters them in-app. This only becomes a problem if TT Personnel ever exceeds ~500 rows. For a cadets register, that won't happen.

Leave this formula as-is. To hide the yellow underline in Studio: File → Settings → Upcoming features → Experimental → toggle Show delegation warnings off. But this is cosmetic only.

Gray buttons with gray text

Visual only — no formula changes needed

⚠ Visual issue — fix by eye

Some buttons have a gray background with gray text, making them invisible. This is a styling issue only — no formula errors involved.

1
Select the button in the Tree View

Click the affected button. Check the right panel — you'll see Fill and Color fields under General.

2
Set the Fill colour

Under Fill, set a visible background. The app's primary blue is #1c3352 — or use the colour picker to pick any clear colour.

3
Set the text colour

Under Color (text colour), set it to white #FFFFFF so it's readable against the new background.

Why it happened Default button colours inherit from a theme or were set explicitly to gray at some point. No formula is involved, so this is purely a visual fix applied in the right panel.
