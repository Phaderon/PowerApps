1

No way to reach the Manage screen

scrAdminDash is missing a button to navigate to scrAdminManage

Missing navigation
What happened The guide built scrAdminDash (compliance dashboard) but never added a button to get to scrAdminManage (where you add courses, add people, etc.). The back arrow on scrAdminDash only goes back to scrMyTraining. scrAdminManage was unreachable.

Add a modern Button directly on scrAdminDash with these properties. It sits in the header bar to the left of the Export PDF button.

PropertyValue
TypeButton (modern)
X986
Y16
Width180
Height40
Text"Manage courses & people"
FontSize11
BasePaletteColorRGBA(230,241,251,1)
ColorRGBA(12,68,124,1)
OnSelectNavigate(scrAdminManage, ScreenTransition.Cover)
After this fix The flow becomes: home → Admin button → scrAdminDash (compliance overview) → Manage courses & people → scrAdminManage (add/edit courses, add/remove people). The back arrow on scrAdminManage returns to scrAdminDash.
2

Staff search field does nothing

galCompliance.Items is set incorrectly — scrAdminDash

Formula set wrong
What happened The galCompliance Items property was set to txtAdminSearch (just the name of the text input control) instead of the full Filter formula that references it. The gallery had no idea what to display or filter.

Select galCompliance in the Tree View on scrAdminDash. Set the property dropdown to Items. Replace whatever is there with this formula:

galCompliance — Items
Items · galCompliance 
Filter(colCompliance,
    (drpTypeFilter.Selected.Value = "All" ||
     StaffType = drpTypeFilter.Selected.Value) &&
    (IsBlank(txtAdminSearch.Text) ||
     txtAdminSearch.Text in Name)
)
After this fix Typing in the search box filters the list instantly as you type — no search button needed. The type filter dropdown also works at the same time. Both filter together.
3

Compliance gallery shows nothing

scrAdminDash · OnVisible · TT Personnel

Data issue — nothing to do with admin status
Why it's empty The screen's OnVisible loads people with Filter('TT Personnel', Active = true). SharePoint boolean columns default to No / false when a record is created. If nobody has had their Active column explicitly ticked to Yes, colAllPeople is empty — so colCompliance is empty — so the gallery shows nothing. Admin status is irrelevant; it's purely the Active column.

Option A — Fix the data in SharePoint (recommended)

This is the right long-term fix. Active is a meaningful field that controls who appears in compliance reports.

1
Open your TT Personnel SharePoint list

Go to your SharePoint site → Lists → TT Personnel (or however you named it).

2
Edit each person's row

Click the row → Edit → find the Active column → tick it to Yes → Save. Do this for yourself and every person who should appear in the compliance dashboard.

3
Go back into the app and refresh

Navigate away from scrAdminDash and back again to re-trigger OnVisible. The gallery should now show everyone with Active = Yes.

Option B — Remove the Active filter from the formula

If you don't want to manage Active at all, change the first line of scrAdminDash → OnVisible to load everyone from TT Personnel unconditionally. Replace the entire OnVisible with this:

OnVisible · scrAdminDash (Active filter removed) 
ClearCollect(colAllPeople, 'TT Personnel');
ClearCollect(colAllRecords, 'TT TrainingRecords');
ClearCollect(colCourses, 'TT Courses');

ClearCollect(
    colCompliance,
    ForAll(
        colAllPeople As P,
        With(
            {
                req: Filter(
                    colCourses,
                    Status.Value = "Active" &&
                    P.StaffType.Value in ApplicableTo.Value
                )
            },
            With(
                {
                    okCount: CountRows(
                        Filter(
                            req As C,
                            With(
                                {
                                    rec: LookUp(
                                        colAllRecords,
                                        Lower(PersonEmail) = Lower(P.Email) &&
                                        CourseID = C.ID
                                    )
                                },
                                !IsBlank(rec.DateCompleted) &&
                                (
                                    C.RecurrenceMonths = 0 ||
                                    DateAdd(rec.DateCompleted, C.RecurrenceMonths, TimeUnit.Months) >= Today()
                                )
                            )
                        )
                    )
                },
                {
                    PersonID:      P.ID,
                    Name:          P.Title,
                    Email:         Lower(P.Email),
                    StaffType:     P.StaffType.Value,
                    LineManager:   P.LineManagerEmail,
                    ManagerEdit:   Coalesce(P.ManagerCanEdit, false),
                    Required:      CountRows(req),
                    InDate:        okCount,
                    Outstanding:   CountRows(req) - okCount,
                    CompliancePct: If(
                                       CountRows(req) = 0,
                                       100,
                                       Round(100 * okCount / Max(CountRows(req), 1), 0)
                                   )
                }
            )
        )
    )
)
Option B trade-off Everyone in TT Personnel will appear in the dashboard regardless of whether they're still active. Fine for now, but you'll want to add Active filtering back once you're managing staff properly and have people leaving.
