#Training Tracker — Power Apps Build Guide

#OVERVIEW

v2.3 audit note: this version was scrubbed for the known Power Fx copy/paste traps and instruction gaps. Any section that previously said “same technique”, “duplicate from prior screen”, or “same formula” has been expanded so you do not have to jump backwards to work out which controls/properties to create.

A complete, copy-paste-ready build plan for a canvas Power App replacing a manual SharePoint training list. Staff maintain their own training records and certificates. Line managers can view (and optionally upload for) their direct/indirect reports. The training administrator ("Jackie") gets a full compliance dashboard, personnel and course management, and one-click PDF report export.

Format: Tablet canvas app, 1366×768. Modern controls enabled.

#What the finished app does

Who What they get
All staff Personal dashboard of mandatory courses for their staff type. Status pills: Not Started, In Date, Expiring Soon, Expired, Complete. Sorted worst-first. Archive section for courses no longer required — records and certificates kept forever.
All staff Open any course, drag-and-drop a PDF certificate, set date of completion manually, save. Replace or delete certificates. Certificate downloads on tap.
Line managers "My Team" tab (visible only if they manage anyone). Shows full reporting chain up to 3 levels, collapsed accordion, compliance mini-bar per person. Read-only by default. Admin can grant upload rights per person.
New starters Holding screen: "No training record found — contact [admin]." Try Again button.
Admin Compliance matrix of all staff, worst-first. Per-person drill-down with manager-edit toggle and audit trail. Headline stats. Export PDF report button.
Admin Full personnel and course management inside the app. M365 people search with auto-fill. Manual entry fallback for staff without M365 accounts. Line manager assignment per person.

#Architecture

Three SharePoint lists. Everything loaded into in-memory collections on screen entry. At 60–70 staff and ~20 courses this is trivially small — keeps formulas fast, simple, and free of delegation warnings.

Component Purpose
TT Personnel Who is in the system. Staff type, admin flag, line manager chain, M365 access flag.
TT Courses Course catalogue. Who it applies to, recurrence in months (0 = one-off), Active/Archived.
TT TrainingRecords One row per person per course. Email key, CourseID, course name snapshot, date completed, uploaded-by audit. Certificates as list attachments.
Canvas app 7 screens: scrStart, scrNoRecord, scrMyTraining, scrCourseDetail, scrTeamMember, scrAdminDash, scrAdminManage.
Power Automate One flow: HTML built by app → OneDrive PDF conversion → email + save to SharePoint.

#Key design decisions

Decision Why
Plain text email keys, no SharePoint lookup columns Lookup columns are painful to Patch and filter. Plain keys joined in-memory are bulletproof at this scale.
Course name snapshotted into each record Archived or deleted courses still read correctly on training records forever.
Expiry calculated, never stored Change a course's recurrence and every status updates instantly — no data migration.
Certificates as list attachments Native drag-and-drop, native delete, zero extra infrastructure, downloads on tap.
varChainDepth = 3 Single variable to increase later. Prevents runaway recursion on bad data.
HasM365Access flag Staff without M365 accounts fall back to manually-entered name. No broken API calls.
UploadedByEmail audit trail When a manager uploads on behalf of someone, the admin can see who did it.

Build phase#PHASE 1 — Build the SharePoint lists

Create all three as blank lists on the same SharePoint site. Create columns with the exact internal names shown — no spaces in any column name (spaces in internal names become Column_x0020_Name in Power Apps formulas, which is a nightmare to work with).

#TT Personnel

Column Type Settings
Title Single line (built-in) Display name. Required.
Email Single line Lowercase M365 email. Required. This is the app's key.
StaffType Choice Choices: Civil Servant | Army | Contractor. Dropdown. Required. No fill-in.
IsAdmin Yes/No Default: No.
Active Yes/No Default: Yes. Deactivating hides the person without deleting history.
LineManagerEmail Single line Optional. Must match another row's Email value. Leave blank for top of chain.
ManagerCanEdit Yes/No Default: No. If Yes, line manager can upload certificates on this person's behalf.
HasM365Access Yes/No Default: Yes. Set No for staff without M365 accounts — falls back to manually-entered name.

Note: Add yourself and the admin now, both with IsAdmin = Yes. Email must exactly match the M365 sign-in address, lowercase.

#TT Courses

Column Type Settings
Title Single line (built-in) Course name. Required.
ApplicableTo Choice Choices: Civil Servant | Army | Contractor. Tick "Allow multiple selections" (checkboxes). Required.
RecurrenceMonths Number 0 decimal places. 0 = one-off, 12 = yearly, 24 = every 2 years. Default 0. Required.
Status Choice Choices: Active | Archived. Default: Active. Required.
CourseInfo Multiline plain text Optional description or link.

#TT TrainingRecords

Column Type Settings
Title Single line (built-in) Auto-filled by app as "Person - Course". Not used in logic.
PersonEmail Single line Lowercase email of the record owner. Required.
CourseID Number 0 decimals. SharePoint ID of the course in TT Courses. Required.
CourseName Single line Snapshot of course name at time of completion. Survives archival/deletion. Required.
DateCompleted Date and time Date only (no time). Optional — blank until person records completion.
UploadedByEmail Single line Optional. Blank if person uploaded themselves. Populated when a manager uploads on their behalf.

Note: Enable attachments on TT TrainingRecords: List settings → Advanced settings → Attachments → Enabled.

#scrMyTraining.OnVisible

Power Fx / formula
ClearCollect(colMyRecords,
    Filter('TT TrainingRecords', PersonEmail = varMyEmail));

ClearCollect(colMyTraining,
    ForAll(
        Filter(colCourses,
            Status.Value = "Active" &&
            varMe.StaffType.Value in ApplicableTo.Value
        ) As C,
        With({rec: LookUp(colMyRecords, CourseID = C.ID)},
        {
            CourseID:    C.ID,
            CourseName:  C.Title,
            CourseInfo:  C.CourseInfo,
            RecurMonths: C.RecurrenceMonths,
            RecurText:   Switch(C.RecurrenceMonths,
                             0,  "One-off",
                             12, "Every year",
                             24, "Every 2 years",
                             "Every " & C.RecurrenceMonths & " months"),
            RecordID:    rec.ID,
            DateDone:    rec.DateCompleted,
            UploadedBy:  rec.UploadedByEmail,
            ExpiresOn:   If(IsBlank(rec.DateCompleted) || C.RecurrenceMonths = 0,
                             Blank(),
                             DateAdd(rec.DateCompleted, C.RecurrenceMonths,
                                     TimeUnit.Months)),
            TrainingStatus:
                If(IsBlank(rec.DateCompleted), "Not Started",
                   C.RecurrenceMonths = 0, "Complete",
                   DateAdd(rec.DateCompleted, C.RecurrenceMonths,
                           TimeUnit.Months) < Today(), "Expired",
                   DateAdd(rec.DateCompleted, C.RecurrenceMonths,
                           TimeUnit.Months) <= DateAdd(Today(), 60,
                           TimeUnit.Days), "Expiring Soon",
                   "In Date")
        })
    )
);

ClearCollect(colMyArchive,
    Filter(
        ForAll(colMyRecords As R,
            {
                RecordID:   R.ID,
                CourseName: R.CourseName,
                DateDone:   R.DateCompleted,
                IsArchived: IsBlank(LookUp(colCourses, ID = R.CourseID, ID)) ||
                            LookUp(colCourses, ID = R.CourseID,
                                   Status.Value) = "Archived"
            }
        ),
        IsArchived && !IsBlank(DateDone)
    )
)

You are converting this markdown file into a polished, professional PDF build guide. Follow these formatting rules precisely — they exist to fix specific problems from a previous version of this document.

If a code block won't fit in the remaining space on the current page, start a new page and place the whole block there.   8. Format target: A4, portrait, professional but not sterile — this is for one specific person (a developer named Allan) building this app himself, not a generic corporate deliverable.

#CRITICAL TECHNICAL NOTES — READ BEFORE FORMATTING

These are hard-won corrections from real build experience. Do not silently "fix" or rephrase any of the following — they are correct as written, even where they may look unusual:

  • Rounded corners: Power Apps rectangles have NO radius property. Every "card" or "rounded panel" in this guide is built using a classic Button control with Text="", DisplayMode=DisplayMode.View, HoverFill and PressedFill locked to match Fill (to prevent hover-darkening), and then RadiusTopLeft/RadiusTopRight/RadiusBottomLeft/RadiusBottomRight set normally. Never suggest using Insert → Rectangle for anything that needs rounded corners.
  • List names: All three SharePoint list names contain spaces and must be wrapped in single quotes in every formula: 'TT Personnel', 'TT Courses', 'TT TrainingRecords'. This is not optional and not a style choice — Power Fx requires it for any identifier containing a space.
  • Sort order: Always the full enum prefix — SortOrder.Ascending or SortOrder.Descending. Never bare Ascending/Descending.
  • Modern vs Classic controls: This app uses modern controls (Button, Text input, Dropdown, Combo box, Toggle) for most interactive elements, but classic controls for Icon, Gallery, and Edit form (the Attachments card for certificate upload only exists in classic forms). Every control in this guide states explicitly which type to insert — preserve that labelling.
  • Modern Text Input placeholder property is Placeholder, not HintText (HintText is the classic-control name only).
  • Modern Toggle's value-reading property is .Checked, not .Value. The property that sets the toggle's caption text is Label, not Text. In this build, set the toggle state with Checked; do not hunt for Default if the modern Toggle properties pane does not show it.
  • Office365Users.UserProfileV2 returns lowercase camelCase fields: .displayName, .jobTitle, .mail — NOT .DisplayName/.JobTitle/.Mail. This is different from Office365Users.SearchUserV2(...).value which returns PascalCase fields: .DisplayName, .Mail. This inconsistency is a genuine quirk of the Power Apps connectors, not an error — preserve the casing exactly as shown in each formula below.
  • Modern Combo box: SelectMultiple now defaults to true in current Power Apps, so it must be explicitly set to false anywhere single-selection is required (e.g. the people-search combo box).
  • All RGBA colour values are written as RGBA(r,g,b,1) rather than ColorValue("#hex") — both work, but RGBA was used throughout for consistency. Keep this format.
